# Problem Solving

Active practice problems, one folder per problem.

## Layout

```
problem-solving/
  {PLATFORM}-{number}-{slug}/
    main.go        ← Go solution (default)
    {variant}/     ← optional: different approach or language
      main.go
```

Naming and scaffold rules: see `.cursorrules` at the repo root.

## Goal

Reach the point where these steps are automatic for any problem:

1. Explain brute force
2. Identify bottleneck
3. Optimize
4. State complexity
5. Implement
6. Test edge cases
7. Refactor
